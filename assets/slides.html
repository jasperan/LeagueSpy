The Scraper

scrapling's StealthyFetcher defeats op.gg's anti-bot stack (Cloudflare, fingerprinting, rate limits) so you don't have to.

# stealth fetch, no detection
page = await StealthyFetcher.fetch(url)
rows = page.css(".game-item")

# extract what matters
champion, kda, result, duration = parse_row(row)
  • Real browser fingerprints (no headless detection)
  • Extracts champion, KDA, win/loss, game duration, queue type
  • Handles region-specific op.gg URLs automatically

Oracle Database

Two tables. Clean separation. Only new matches get announced.

SUMMONERS
summoner_idNUMBER PK
game_nameVARCHAR2
tag_lineVARCHAR2
regionVARCHAR2
is_smurfNUMBER(1)
main_account_idNUMBER FK
MATCHES
match_idVARCHAR2 PK
summoner_idNUMBER FK
championVARCHAR2
kdaVARCHAR2
resultVARCHAR2
durationVARCHAR2
scraped_atTIMESTAMP

Diff logic: INSERT only when match_id not in DB. No duplicates, no re-announcements.

Three Steps to Deploy

Create Bot

Register at Discord Developer Portal. Grab the token. Invite to your server with message permissions.

Configure

Add summoner names, regions, smurf links, and DB credentials to config.yaml

Run

python -m src.bot

Scraping starts automatically. Matches flow into Discord within 5 minutes.
